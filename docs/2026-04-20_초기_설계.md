# Pickz 초기 설계

작성일: 2026-04-20

## 1. 프로젝트 개요

- 서비스명: Pickz
- 한정판 굿즈를 대상으로 하는 실시간 경매 서비스
- 핵심 목표: 대규모 동시 입찰 환경에서 정합성과 확장성을 보장하는 백엔드 설계 구현

## 2. 기술 방향

- Java 17/21, Spring Boot 3.x
- Spring Data JPA, MySQL
- Redis(Redisson) 기반 분산 동시성 제어
- WebSocket 기반 실시간 입찰가 반영
- Spring Security + JWT 인증/인가

## 3. 요구사항 초안 (MVP 중심)

### P0 (필수)

- 회원가입/로그인(JWT)
- 경매 상품 등록/조회/상세 조회
- 입찰 기능(최소 입찰 단위, 종료 시간 검증)
- 동시 입찰 충돌 시 원자적 처리로 최고 입찰가 정합성 보장
- 경매 종료 및 낙찰자 확정

### P1

- 자동 연장(스나이핑 방지)
  - 종료 임박 구간(N초) 내 입찰 발생 시 M초 연장
  - 경매별 최대 연장 시각 설정 가능
  - 예시: 기본 마감 16:00, 최대 연장 19:00
  - `new_end_time = min(current_end_time + M, max_end_time)` 규칙 적용

### 제외 범위(현재 단계)

- 알림 기능(푸시/이메일/인앱) 제외
- 결제/배송/정산 제외
- 관리자 기능 고도화 제외

## 4. 핵심 비즈니스 규칙

- 입찰가는 현재 최고가 + 최소 입찰 단위 이상이어야 한다.
- 경매 시작 전, 종료 후에는 입찰할 수 없다.
- 경합 상황에서도 최고가는 단일 흐름으로만 갱신되어야 한다.
- 자동 연장은 최대 연장 시각을 넘길 수 없다.
